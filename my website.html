My Project Idea

Overview

My project involves the idea of a Chat bot for RMIT related questions. While there is an FAQ, there is only a few questions on the page, and doesn't cover nearly as much questions as there are being asked. As time progresses and more questions are asked, the bot will be updated with all of the new information. As we have already created a chat bot, this project should be much easier than any other java related projects that I could have chosen.

Motivation

I feel that this project will be of use as I had trouble with my student card at the start of the year and I noticed the waiting queue for RMIT Connect was pretty large, suggesting they were busy. If this system were in place, it could save both the user and the people at RMIT connect time. It will also mean that the same questions will be repeated a lot less often.

Description

If the Chatbot were to be a success it could potentially be a part of the RMIT website.

It will start off with a user asking a question to RMIT Connect. Then, RMIT connect will attempt to answer the users question to the best of their ability. If RMIT Connect is able to successfully answer the users question, the answer and the question will be added to a database alongside other questions and answers.

To prevent answers stacking up, the person at RMIT Connect will have the option on whether or not to add the question to the database. If it is a unique question and they do wish to add it to the database, all they will have to do is press a button that comes up after the chat with the student has ended. Now if another user has the same question, they can use the Chat bot and type it in. Upon pressing enter, the bot will scan the database for relevant key words, then the user will be granted with every question that has the given Key word in it, such as 'Student Card'.

The bot will list every question with that Key word in a response box, alongside a number.

EG. 1. How do I get my student card?

To choose the question that is most relevant to the user, they will have to type the corresponding number to the question. When the user does this, the bot will display the answer to the question the user was looking for, once again in a response box. If the user types in a question but no Key words are recognized, the bot will tell the user “Sorry, but we do not recognize your Question. Please seek further help on RMIT Connect.“ If the user is unsure how the bot works, there will be a Button labelled '?'. When pressed or hovered, it will bring up a message which reads “Hello! Please type your question in the input box, and we will scan our database to see if this question has been answered before. The Chat bot will aim to be accessible on all sorts of devices and screens. There will need to be someone who checks the database regularly, as questions are likely to repeat, despite the button where the user decides whether or not to add the query to the database.

Tools and Technologies

For the Chatbot itself, Java will be used as I am familiar with it, and it is likely the most optimal programming language for this situation. All of the data that is gathered from usage of RMIT Connect and the Chat bot will be stored in Oracle as a Relational Database. HTML will also be used as the Chat bot will be running on the RMIT website. The data for the database will be stored on a hard drive, and backed up on a cloud server to ensure that the risk of data loss is minimized.

Skills Required

There are quite a few skills needed for this project to work optimally. Skill in Java will be needed, as the Chat bot will be complicated due to all of the possible entries from the user. Knowledge in Relational Databases and Oracle is also necessary, as there will be a lot of data to manage, and it will be regularly updated. It will be quite difficult to learn these skills, as java can be quite complicated, as can large relational databases storing large amounts of updating information.

Outcome

The hopeful outcome of this project is that there will be an easier, new way for the user to find out the solutions to the problems they are having. If the Chat bot functions correctly, it will ensure that RMIT Connect can answer more questions and be less busy, and it will also ensure that the user saves time and will not have to wait in a queue just to resolve a question that has already been answered. To summarize, the Chat bot is an attempt to reduce the traffic on the RMIT Connect website.
